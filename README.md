# Altos Founder Pipeline Report

Internal HTML dashboard for triaging founder applications. Each card is a founder who applied + completed the Turing AI interview, with structured fields (problem, solution, traction, cap table, transcript) extracted by the interview pipeline.

**Aesthetic:** editorial monochrome matching altos.vc.
**Refresh model:** local cron → regenerates `data.json` → `git push` → Vercel auto-redeploys from `main`.

## Layout

```
.
├── index.html             # the report (vanilla JS + CSS, no build)
├── data.json              # snapshot of the founder pipeline (regenerated by cron)
├── bin/
│   ├── refresh.py         # runs query.sql against $DATABASE_URL, writes data.json
│   └── requirements.txt   # psycopg
├── refresh-and-push.sh    # cron entrypoint: refresh + commit + push
├── query.sql              # the SQL (parameterized by :since, defaults to last 30 days)
├── vercel.json            # static-site cache headers
└── deploy/
    ├── cron.example       # cron / launchd snippets
    └── upload.sh          # legacy S3/GCS path (only if you leave Vercel)
```

## Deploy flow

```
Postgres (altos_application + altos_attempts + meeting)
       │
       │  refresh.py  (cron, on your machine)
       ▼
   data.json   ──►   git push   ──►   GitHub main   ──►   Vercel
                                                            │
                                                            ▼
                                                       https://...vercel.app/
```

## First-time setup

```bash
# 1. Clone + Python deps
git clone https://github.com/quynguyen2303/altos_report.git
cd altos_report
python3 -m venv .venv
.venv/bin/pip install -r bin/requirements.txt

# 2. Put DATABASE_URL in .env (gitignored)
echo 'DATABASE_URL=postgres://...' > .env

# 3. Sanity check — refresh + serve locally
.venv/bin/python bin/refresh.py
python3 -m http.server 8765
open http://localhost:8765/

# 4. Connect to Vercel (one time)
npm i -g vercel
vercel link --yes
# Vercel will auto-redeploy on every push to main, no env vars needed
# (the report is served as static files; DB lives on your refresh host only).

# 5. Install the cron (see deploy/cron.example for snippets)
crontab -e
# add:  0 * * * * cd /Users/quy/claude/altos_report && ./refresh-and-push.sh >> /tmp/altos_refresh.log 2>&1
```

## How refresh-and-push.sh works

1. `git pull --rebase --autostash` so concurrent edits to index.html don't clobber.
2. Runs `refresh.py` → writes a new `data.json`.
3. If the diff is empty, exits 0 silently.
4. Otherwise commits `data: refresh <ISO timestamp> (<N> founders)` and pushes.

Vercel watches `main` and rebuilds automatically (typically < 30s).

## Local dev

```bash
.venv/bin/python bin/refresh.py    # regenerate data.json
python3 -m http.server 8765    # serve
open http://localhost:8765/
```

## Security note

This report exposes founder PII (email, phone, passport/visa status, cap table). The Vercel URL is unlisted — keep it that way. If anyone outside the Altos team needs access, gate it with Vercel Password Protection (Pro) or move it behind SSO first.
